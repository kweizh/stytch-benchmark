## Stytch Client Initialization

```js
const client = new stytch.Client({
  project_id: process.env.STYTCH_PROJECT_ID,
  secret: process.env.STYTCH_SECRET,
});
```
The SDK automatically selects the test or live environment based on the `project-test-*` / `project-live-*` prefix of the project ID.

## Endpoints

| Method | Path | Request Body | Response |
|--------|------|-------------|----------|
| `POST` | `/users` | `{ "email": "..." }` | `{ "user_id": "..." }` |
| `POST` | `/totp/enroll` | `{ "user_id": "..." }` | `{ "totp_id": "...", "secret": "..." }` |
| `POST` | `/totp/verify` | `{ "user_id": "...", "totp_code": "..." }` | `{ "session_token": "..." }` |

### Endpoint details
- **`POST /users`** — Calls `client.users.create({ email })` and returns the `user_id`.
- **`POST /totp/enroll`** — Calls `client.totps.create({ user_id })` and returns `totp_id` and `secret` (the base32-encoded TOTP secret from Stytch).
- **`POST /totp/verify`** — Calls `client.totps.authenticate({ user_id, totp_code, session_duration_minutes: 60 })` and returns `session_token`.
