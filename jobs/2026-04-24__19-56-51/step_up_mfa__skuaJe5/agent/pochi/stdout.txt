## `POST /transfer` Logic

| Condition | Status | Response |
|---|---|---|
| Invalid/missing `session_token` (Stytch throws) | `401` | `{"error": "Unauthorized"}` |
| Valid session, `authentication_factors.length < 2` | `403` | `{"error": "Step-up MFA required"}` |
| Valid session, `authentication_factors.length >= 2` | `200` | `{"success": true}` |

## Key Implementation Details
- **Stytch B2C Client** initialized with `STYTCH_PROJECT_ID` and `STYTCH_SECRET` environment variables.
- `client.sessions.authenticate({ session_token })` validates the token — any error (expired, invalid, missing) triggers a `401`.
- The number of authentication factors is checked via `response.session.authentication_factors.length`.
- Server listens on **port 3000**.
