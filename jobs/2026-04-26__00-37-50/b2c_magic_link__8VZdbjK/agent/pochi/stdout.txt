## Implementation Details (`index.js`)

**Stytch Client Initialization:**
- Reads `STYTCH_PROJECT_ID` and `STYTCH_SECRET` from environment variables
- Creates a `stytch.Client` instance used across both routes

**`POST /login`:**
- Reads `email` from the JSON request body
- Calls `stytchClient.magicLinks.email.loginOrCreate()` with both `login_magic_link_url` and `signup_magic_link_url` set to `http://localhost:3000/authenticate`
- Returns `{ message: "Magic link sent successfully" }` on success or a `500` on failure

**`GET /authenticate`:**
- Reads the `token` query parameter from the URL
- Calls `stytchClient.magicLinks.authenticate({ token })`
- On success: returns `{ "success": true, "user_id": "<stytch_user_id>" }`
- On failure: returns `401` with `{ "success": false, "error": "Authentication failed" }`

**Server listens on port `3000`.**
