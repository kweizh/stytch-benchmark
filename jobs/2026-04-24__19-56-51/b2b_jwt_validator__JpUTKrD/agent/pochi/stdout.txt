## What was built

### `POST /validate`
- Accepts `{ "session_jwt": "<token>" }` in the JSON body.
- Returns `200 { "member_id": "..." }` on a valid token.
- Returns `400` for a missing/non-string payload.
- Returns `401` for an invalid or expired JWT.

### Local validation — no per-request network call
The service uses `stytchClient.sessions.authenticateJwtLocal()` from the official `stytch` SDK (`^14`). On first use the SDK fetches and **caches** the project's JWKS in memory. All subsequent calls verify the JWT signature and expiry purely in-process. The SDK re-fetches JWKS automatically only when it encounters an unknown `kid`, so JWKS key rotations are handled transparently.

### Configuration
```
STYTCH_B2B_PROJECT_ID=project-live-...
STYTCH_B2B_SECRET=secret-live-...
```
The process exits with a clear error message if either variable is missing.
