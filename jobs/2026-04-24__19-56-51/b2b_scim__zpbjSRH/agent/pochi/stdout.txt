### How `setup_scim.js` works

```
STYTCH_PROJECT_ID + STYTCH_SECRET
        │
        ▼
stytch.B2BClient({ project_id, secret })
        │
        ├─► organizations.create({
        │       organization_name: "Acme SCIM Org",
        │       organization_slug: "acme-scim-org-<timestamp>"   ← unique per run
        │   })
        │        └─► organization_id
        │
        └─► scim.connection.create({
                organization_id,
                display_name:      "Acme SCIM Connection",
                identity_provider: "okta"
            })
                 └─► connection_id, base_url, bearer_token
                              │
                              ▼
                  scim_output.json  {
                      organization_id,
                      connection_id,
                      base_url,
                      bearer_token          ← only returned at creation time
                  }
```

Key design decisions confirmed against the SDK type definitions:
- `client.organizations.create` → returns `{ organization: { organization_id, … } }`
- `client.scim.connection.create` → returns `{ connection: SCIMConnectionWithToken }` where `SCIMConnectionWithToken` exposes `bearer_token` **only at creation time** (subsequent calls return only `bearer_token_last_four`)
- Credentials are read exclusively from `STYTCH_PROJECT_ID` / `STYTCH_SECRET` environment variables; nothing is hardcoded
