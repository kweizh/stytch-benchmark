## `login.js` — How it works

1. **Reads `telemetry_id`** from `process.argv[2]`; exits with code 1 and a usage message if missing.
2. **Reads credentials** from `STYTCH_B2B_PROJECT_ID` and `STYTCH_B2B_SECRET` environment variables; exits with code 1 if either is absent.
3. **Initializes `B2BClient`** with those credentials.
4. **Calls `client.fraud.fingerprint.lookup({ telemetry_id })`** — the SDK's B2B client exposes `fraud.fingerprint` backed by the Stytch Fraud API (`POST /v1/fingerprint/lookup`).
5. **Evaluates `verdict.action`**:
   | `verdict.action` | Output | Exit code |
   |---|---|---|
   | `"BLOCK"` | `Access Denied: Blocked` | 1 |
   | `"CHALLENGE"` | `MFA Required` | 1 |
   | `"ALLOW"` | `Login Successful` | 0 |
6. **Error handling**:
   - A 404 / `telemetry_id_not_found` error is treated as a `BLOCK` (per Stytch's recommendation for attacker-controlled IDs).
   - All other API/network errors print a descriptive message and exit with code 1.
