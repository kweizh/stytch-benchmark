### How it works (`mcp_auth.js`)

```
STYTCH_PROJECT_ID + STYTCH_SECRET (env vars)
        │
        ▼
  stytch.B2BClient  ← initialised once at module load
        │
        ▼
  verifyAgentToken(jwt)
        │
        ├─ guards: jwt must be a non-empty string
        │
        ├─ client.sessions.authenticateJwt({ session_jwt: jwt })
        │      • local JWKS verification (no network on cache-hit)
        │      • throws StytchError on invalid / expired / revoked JWT
        │
        └─ returns  response.member_session  ✓
```

**Key design decisions:**

- **`B2BClient.sessions.authenticateJwt`** — verifies the JWT locally against the cached JWKS (fast path), falling back to a remote call only when the signing key isn't yet cached. This avoids a synchronous network round-trip on every request.
- **Module-level client** — the `B2BClient` is constructed once and reused, preserving the JWKS cache and connection pool across calls.
- **Fail-fast on missing env vars** — the module throws at `require()` time rather than silently failing at runtime when credentials are absent.
- **Error propagation** — the Stytch SDK throws a `StytchError` (with a descriptive `error_type` field) for any auth failure; `verifyAgentToken` lets that bubble up so callers can decide on the HTTP response / logging strategy.

### Usage

```js
const { verifyAgentToken } = require('./mcp_auth');

// In your MCP request handler:
try {
  const memberSession = await verifyAgentToken(req.headers['x-session-jwt']);
  // memberSession.member_id, memberSession.organization_id, etc.
} catch (err) {
  // err is a StytchError or plain Error
  res.status(401).json({ error: err.message });
}
```
