### `mcp_auth.js` — design decisions

```
verifyAgentToken(jwt)
  │
  ├─ Guard: throws immediately if jwt is null / empty / non-string
  │
  ├─ Lazy-initialises a shared stytch.B2BClient singleton
  │    reads STYTCH_PROJECT_ID and STYTCH_SECRET from env
  │    throws descriptively if either variable is missing
  │
  ├─ Calls client.sessions.authenticateJwt({ session_jwt: jwt })
  │    • validates JWT signature via Stytch's JWKS (local, no extra RTT)
  │    • confirms session is not revoked (remote check)
  │
  └─ Returns response.member_session on success
     Propagates StytchError / Error on any failure
```

**Why `authenticateJwt` instead of `authenticateJwtLocal`?**  
`authenticateJwt` performs a hybrid check — it verifies the signature locally via JWKS _and_ hits the Stytch API to confirm the session has not been explicitly revoked. `authenticateJwtLocal` skips the revocation check, which is acceptable for low-latency paths but less safe for an MCP auth gate.

### Usage

```js
const { verifyAgentToken } = require("./mcp_auth");

// In your MCP request handler:
app.use(async (req, res, next) => {
  const jwt = req.headers["x-session-jwt"];
  try {
    const session = await verifyAgentToken(jwt);
    req.memberSession = session;   // { member_id, organization_id, … }
    next();
  } catch (err) {
    res.status(401).json({ error: "Unauthorized", detail: err.message });
  }
});
```

