## Key implementation details

- **Stytch B2C client** — `new stytch.Client({ project_id, secret })` reads from `STYTCH_PROJECT_ID` and `STYTCH_SECRET` env vars; the SDK automatically selects test vs. live environment from the project ID prefix.
- **`POST /users`** — calls `client.users.create({ email })` and returns `response.user_id`.
- **`POST /totp/enroll`** — calls `client.totps.create({ user_id })` and returns `response.totp_id` + `response.secret` (the base32 secret is a top-level field on the Stytch response).
- **`POST /totp/verify`** — calls `client.totps.authenticate({ user_id, totp_code, session_duration_minutes: 60 })` and returns `response.session_token`.
- Server listens on **port 3000**.
